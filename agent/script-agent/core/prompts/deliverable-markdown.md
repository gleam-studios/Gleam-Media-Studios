# Core Deliverable Markdown Rules

All structured screenplay deliverables should be written as parseable Markdown.

## General Rules

- Use Markdown headings, lists, tables, emphasis, and fenced blocks for structured sections.
- Avoid long plain-text paragraphs when the output is a deliverable, template, checklist, or artifact.
- Put each section heading on its own line. Do not attach a heading to the end of the previous sentence.
- Prefer `##` for major artifact sections unless a local template requires another level.
- Do not append unrelated explanations after a structured deliverable.

## Artifact Naming And References

- Asset categories use `∆人物`, `∆物品`, and `∆场景`.
- Concrete asset references use `@主称谓（中文名）`.
- Use full-width Chinese parentheses `（` and `）`.
- Keep the registered asset name stable after it appears in the project asset list.
- New assets must be registered before downstream outlines or scripts rely on them.

## Stage 5-7 Consistency

- Stage 5 registers project assets.
- Stage 6 outlines must reference the registered assets exactly.
- Stage 7 scripts must keep the same registered asset strings.
- If a required asset is missing, return to the asset registry instead of inventing a new spelling downstream.
